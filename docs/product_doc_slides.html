项目简介

PRODUCT OVERVIEW

AlphaInvestPro 是一款简洁的智能投资分析产品（本地优先 MVP）。用户输入一家公司或股票代码，选择分析深度， 系统即通过 AI 智能体工作流 产出一份覆盖 商业质量、护城河、估值、风险、多空论点与最终结论 的结构化投资备忘录。

不是聊天机器人

不把推理藏在长篇对话里。每一步分析——快速筛查、质量透镜、估值检查、逆向风险审查、最终备忘录——都以可见的时间线呈现。

纪律化价值投资框架

内置经典价值投资分析范式：商业质量与护城河评估、管理层与资本配置审视、估值假设透明化、逆向思维式风险清单。

仅限研究用途

定位为投资研究工具，不含任何交易执行、券商接入或自动化下单能力。所有输出附带风险与不确定性说明。

中英双语界面（English / 简体中文） 结构化 JSON → Markdown 报告 可视化智能体流水线 本地优先部署 非投资建议 · Research Only

解决的问题与应用场景

PROBLEM & USE CASES

用户痛点

  • 通用 AI 聊天工具产出的研报冗长、不稳定、不可复现；
  • 商业质量、估值、风险与最终结论混在一起，缺乏分析纪律；
  • AI 可能"编造"财务数字，数据来源与假设不透明；
  • 普通投资者缺少一套像分析师检查清单一样的结构化工作流。

产品承诺

  • 一次输入，产出结构统一的投资备忘录；
  • 分析步骤全程可见、可审计、可导出；
  • 所有财务数字来自真实数据源，标注来源与时间戳；
  • 每个估值数字都暴露背后假设，每个结论都附带风险提示。

核心功能说明

CORE FEATURES

① 四种分析模式

  • 快速筛查（Quick Screen）
  • 完整备忘录（Full Memo）
  • 风险审查（Risk Review）
  • 估值检查（Valuation Check）

② 可视化分析时间线

实时展示每个智能体步骤的进度与产出：快速筛查 → 商业质量 → 护城河 → 管理层与资本配置 → 估值 → 逆向风险 → 最终备忘录。

③ 结构化投资备忘录

输出统一 Schema：最终判断（回避 / 观察 / 有吸引力 / 不确定）、置信度、质量与护城河评分、估值区间、多空论点、风险清单。

④ 估值假设表

支持简单倍数法、轻量 DCF、FCF 收益率等方法；每条假设均标注名称、取值与数据来源，给出低 / 基准 / 高三档合理价值区间。

⑤ 逆向风险审查

逆向思维式提问："什么情况会让这笔投资失败？" 输出核心风险、论点杀手（thesis killers）与逆向检验问题。

⑥ 报告历史 / 自选股 / 设置

报告可保存、检索与导出；自选股列表沉淀观察标的；设置页展示 LLM 连接状态与语言切换（English / 简体中文）。

AI 使用方式与技术方案（一）

AI ARCHITECTURE — ROLE OF AI

AI 是本产品的核心功能而非辅助功能：整条"分析流水线"由大语言模型驱动的多步骤智能体工作流构成。 同时产品坚持一条铁律——LLM 负责推理与写作，确定性代码负责计算与数据。

AI 能力在产品中的具体作用
大语言模型（LLM）统一采用 DeepSeek（默认 deepseek-chat）作为唯一 LLM 运行时，经后端 DeepSeekClient 封装调用，密钥仅存于服务端。
智能体工作流（Agentic Workflow）分析编排器按固定顺序调度多个分析步骤（筛查 → 质量 → 护城河 → 管理层 → 估值 → 逆向风险 → 备忘录），每步产出独立、可检查的中间结果。
工具调用（Tool Calling）LLM 通过"发现 → 检查 → 调用"流程使用金融数据网关的能力（行情、财报、公司画像、新闻情绪），而非凭空生成数据。
提示词工程（Prompt Engineering）将价值投资方法论（质量透镜、逆向思维审查、估值纪律、风险闸门）固化为分步骤的系统提示词模板，保证输出风格与结构稳定。
结构化输出（Structured Output）所有分析结果先约束为严格 JSON Schema（评分、区间、证据数组），再渲染为 Markdown 备忘录，保证可校验、可存储、可复现。
文本理解 / 生成理解公司业务描述与新闻情绪作为辅助证据；生成中 / 英双语备忘录，跟随界面语言输出。

AI 使用方式与技术方案（二）

AI ARCHITECTURE — TECH STACK & GUARDRAILS

技术栈

  • 前端：Next.js + TypeScript + Tailwind + shadcn/ui，i18n 双语切换；
  • 后端：FastAPI + Python，分析编排器 + 服务分层（LLM 服务 / 行情服务 / 基本面服务 / 报告服务 / 证据服务）；
  • 数据库：SQLite（MVP），抽象层可平滑迁移 PostgreSQL；
  • 实时进度：Server-Sent Events（SSE）推送智能体步骤事件；
  • 报告渲染：结构化 JSON → Markdown（后续支持 PDF）。

AI 安全护栏（Guardrails）

  • 禁止编造数字：价格、财报、估值输入、市值、比率等一律来自数据层，LLM 只能引用；
  • 确定性计算：估值等金融计算由代码完成，附假设、来源、时间戳与校验；
  • 可审计：每个 AI 输出可解释、可回看、有日志、可导出；
  • 降级策略：LLM 密钥缺失时功能明确降级，界面清晰标注 MOCK LLM OUTPUT；
  • 数据完整性闸门：关键财务字段缺失时拒绝生成"完整"报告，或显式标记为不完整。

数据来源与数据处理方式

DATA SOURCES & PIPELINE

统一数据来源：QVeris.ai 金融数据网关

所有股票金融数据经由 QVeris.ai 统一接入（后端专用 QverisClient 封装），覆盖：

  • 实时报价与最新价格、历史价格趋势；
  • 公司画像与业务描述；
  • 财务报表与归一化基本面数据；
  • 估值输入项（营收、利润、自由现金流、股本、负债、现金、利润率、比率）；
  • 市场新闻与情绪——仅作辅助证据，不作最终决策依据。

数据处理与治理

  • 归一化：全部外部响应先转换为内部统一 Schema，再进入分析智能体与 UI；
  • 全链路溯源：每个数据点记录提供方、能力 ID、源时间戳、获取时间戳、币种、代码、市场与数据质量备注；
  • 密钥安全：API Key 仅存于服务端环境变量，绝不进入前端、日志、报告或代码仓库；
  • 降级机制：数据源不可用时仅回退到显式演示数据，界面与报告明确标注 MOCK DATA；
  • 校验闸门：估值表必须展示数据来源、时间戳与人工假设，缺失关键字段的报告将被拒绝或标记不完整。

项目创新点

INNOVATIONS
  • 可见的智能体工作流，而非"黑箱 AI 魔法" —— 用户能看到每个分析步骤的输入、产出与证据，把 AI 投研从"信不信"变成"查得到"。
  • "LLM 推理 + 确定性计算"双轨架构 —— 金融数字全部由代码计算并附假设与来源，LLM 只做定性推理与写作，从机制上杜绝 AI 编造财务数据。
  • 价值投资方法论产品化 —— 将质量透镜、逆向思维审查、估值纪律、风险闸门等经典分析范式固化为可复用、可复现的工作流，而非一次性提示词演示。
  • 结构化输出优先 —— 统一 JSON Schema 先行、Markdown 渲染在后，报告可校验、可比较、可沉淀为研究资产。
  • 全链路数据溯源与降级透明 —— 每个数据点带来源与时间戳；任何 Mock 数据 / Mock 输出都在界面显式标注，保持诚实。
  • 克制的产品边界 —— 明确"仅限研究、绝不下单"，把研究判断与交易执行彻底分离，是负责任 AI 金融产品的设计范本。

风险提示、局限性与未来改进方向

RISKS & ROADMAP

风险提示

  • 非投资建议：所有输出仅供研究参考，不构成任何买卖建议；
  • AI 输出可能存在错误、偏差或遗漏；
  • 估值假设可能不成立，合理价值区间不代表未来价格；
  • 产品不承诺任何盈利能力，投资决策责任在用户。

当前局限性

  • MVP 阶段，依赖单一 LLM 供应商与单一数据网关；
  • 数据覆盖与质量受上游数据源限制；
  • 估值方法偏简化（倍数 / 轻量 DCF），不含复杂建模；
  • 无回测、无组合管理、无多租户与生产级安全承诺；
  • 本地优先部署，暂无云端 SaaS 版本。

未来改进方向

  • 报告 PDF 导出 与更丰富的可视化图表；
  • 行业对比与多标的横向分析；
  • 引入向量记忆 / RAG，沉淀历史研究知识库；
  • PostgreSQL 迁移与多用户支持；
  • 智能体评估回路（Agent Evals），持续度量分析质量；
  • 更多估值方法与情景敏感性分析。
设计原则始终不变：研究与执行分离——无论未来如何演进，产品都不会加入实盘交易、券商接入或自动下单能力； 所有 AI 结论必须继续保持可解释、可审计、附带风险与不确定性说明。
